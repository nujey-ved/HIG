# Onboarding

---

<aside>

온보딩은 앱을 빠르게 사용하도록 지원할 수 있습니다. 

</aside>

- 경험을 통해 앱을 알아 가는 것이 바람직하지만, 온보딩이 필요한 경우라면 빠르고 재미있으며 선택이 가능한 방식으로 디자인하여 제공
- 실행 완료 후 온보딩 제공
    - 실행 경험의 일부가 아니라 별도의 시퀀스

---

#### 모범 사례

- 상호작용
    - 단순 자료 제공보다는 상호작용을 통해서 정보를 파악할 수 있도록
    - 대화식 온보딩 경험으로
- 단일 온보딩 흐름 대신 상황별 팁 모음 제공
    - 새로운 정보를 접하기 전에 제공 → 해당 상황에 집중 → 학습 효과 ⬆️
- 온보딩 경험은 빠르고 재밌게
    - 너무 많은 정보를 단순히 주입 → 부담 & 기억하지 못할 가능성 높음
- 튜토리얼 → 옵셔널
    - skip 처리가 되면 다시 노출되지 않게
    - 사용자가 다시 원할경우에는 접근 가능하게

#### 추가 컨텐츠

- 필요한 경우 시작 화면을 간략하게 표시
    - 시작 화면이 오래 시간을 차지 → 경험이 늦어진다는 느낌
- 대량 다운로드 → 온보딩에 방해되지 않도록
    - 온보딩은 앱을 시작하자마자
    - 다운로드 완료까지 기다릴 필요 없이
- 온보딩 과정에 사용권 세부사항이 표함되지 않도록

#### 추가 요청

- 필요하지 않은 설정이나 사용자화 단계는 나중에
    - 바로 앱과 상호작용 가능하도록 합리적인 수준으로 구성
- 비공개 데이터 또는 리소스에 접근해야 동작이 가능한 경우 온보딩 과정에 권한 요청을 통합
    - 권환 부여에 따른 이점을 보여줘서 타당성
- 앱 경험 후 평가나 구매를 요청
    - 충분한 경험 후 이런한 요청에 긍정적으로 응답할 가능성 높음
